## Plagiarism Prevention

### What We Monitor
- Duplicate repository forks without attribution
- Stolen code published as original
- System prompts republished without credit
- Brand/naming misuse (KOO-, -SUPERDEV)

### Detection & Enforcement
- 🔍 Regular monitoring of GitHub/npm
- 📧 DMCA takedown for violations
- ⚖️ Legal action for commercial theft
- 🤝 Friendly outreach for honest mistakes

### Reporting Violations

If you find KOO Superdev code or content being used without proper attribution:

**Email**: copyright@koo-superdev.dev  
**GitHub Issue**: [Report here](https://github.com/koo/koo-superdev/issues)

Include:
- 🔗 Link to infringing content
- 📷 Screenshots/evidence
- 📝 Description of violation
- 👤 Contact information

## Safe Fork & Contribution Guide

### Creating a Legitimate Fork

✅ **DO THIS:**
```bash
# Fork on GitHub
git clone https://github.com/YOUR_USERNAME/koo-superdev.git
cd koo-superdev

# In README.md, add:
> This is a fork of [KOO Superdev](https://github.com/koo/koo-superdev)
> with [description of changes]

# Make changes
git add .
git commit -m "feat: my changes"

# Push to your fork
git push origin feature/my-feature

# Create Pull Request
```

✅ **DO NOT:**
- ❌ Remove copyright headers
- ❌ Change package name without prefix (e.g., use `koo-superdev-custom`, not `superdev`)
- ❌ Hide the original repository link
- ❌ Claim code as your own creation
- ❌ Use "KOO" branding without attribution
